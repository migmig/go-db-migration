# DBMigrator Documentation

## Directory Structure

- `vXX/`: Active or current development cycle documentation (e.g., `v26/`).
- `complete/`: Archive of all completed development cycles.
    - `HISTORY.md`: Consolidated technical history and PRDs of all past versions.

## Archive Policy
Once a version is fully implemented and verified, its `prd.md`, `spec.md`, and `tasks.md` should be appended to `complete/HISTORY.md` and the individual folder removed to keep the workspace clean.
